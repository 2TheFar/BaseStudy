# 图中的 DFS 和 BFS

图里的 DFS 和 BFS，本质都是从一个点出发，把和它有关联的点访问一遍。

区别在于：

```text
DFS：一条路走到底
BFS：一圈一圈往外扩散
```

## DFS：深度优先搜索

DFS 的感觉是：

```text
先选一个方向一直走
走不动了再回退
换另一个方向继续走
```

比如「岛屿数量」这道题里，从一个 `'1'` 出发，把它上下左右相连的所有 `'1'` 都处理掉。

它特别适合解决：

```text
这一片连通区域有多大？
这一整块东西能不能全部访问到？
有多少个连通块？
从起点能不能走到终点？
```

典型题：

```text
200. 岛屿数量
695. 岛屿的最大面积
130. 被围绕的区域
797. 所有可能的路径
```

DFS 的常见写法：

```java
private void dfs(int row, int col) {
    if (越界 || 已访问 || 不符合条件) {
        return;
    }

    标记当前位置已访问;

    dfs(row + 1, col);
    dfs(row - 1, col);
    dfs(row, col + 1);
    dfs(row, col - 1);
}
```

它的核心是：

```text
进入一个点之后，立刻继续深入它的邻居。
```

## BFS：广度优先搜索

BFS 的感觉是：

```text
先访问离我最近的一圈
再访问下一圈
再访问再下一圈
```

它像水波扩散，也像腐烂橘子感染。

比如「腐烂的橘子」：

```text
第 0 分钟：原本腐烂的橘子
第 1 分钟：它们周围的新鲜橘子腐烂
第 2 分钟：上一分钟腐烂的橘子继续感染
```

所以 BFS 特别适合：

```text
最短路径
最少步数
最少分钟数
按层扩散
多源同时出发
```

典型题：

```text
994. 腐烂的橘子
102. 二叉树层序遍历
127. 单词接龙
1091. 二进制矩阵中的最短路径
```

BFS 的常见写法：

```java
Queue<int[]> queue = new LinkedList<>();
queue.offer(new int[]{startRow, startCol});

while (!queue.isEmpty()) {
    int size = queue.size();

    for (int i = 0; i < size; i++) {
        int[] cur = queue.poll();

        处理当前位置;

        把符合条件的邻居加入队列;
    }

    层数 / 步数 / 分钟数++;
}
```

它的核心是：

```text
队列先进先出，保证先处理近的，再处理远的。
```

## 怎么判断用 DFS 还是 BFS

如果题目问：

```text
有几个？
能不能连通？
把一整块区域处理掉？
```

优先想 DFS。

如果题目问：

```text
最短多少步？
最少多少分钟？
扩散几层？
从多个源头同时开始？
```

优先想 BFS。

## 两道题的对比

```text
岛屿数量：DFS，把整座岛沉掉
腐烂橘子：BFS，一分钟一层往外腐烂
```

一句话记：

```text
DFS 是钻洞，一条路走到底。
BFS 是涨潮，一层一层淹过去。
```
